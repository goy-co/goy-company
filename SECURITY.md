# Security Policy

## Reporting a Vulnerability

The Goy Company takes the security of our infrastructure and our users' data very seriously. If you find a security vulnerability in any of our applications or packages, we appreciate your help in reporting it responsibly.

**Please do not open a public GitHub issue for security-related bugs.**

### How to report
Please send an email to **security@goycompany.com** with the following information:
- A description of the vulnerability.
- Steps to reproduce (including scripts or screenshots if possible).
- The potential impact of the vulnerability.

### Our Commitment
We will acknowledge receipt of your report within **48 hours** and provide a timeline for resolution. We ask that you keep the details of the vulnerability confidential until we have had a chance to fix it.

---

## Supported Versions

Only the latest version of the `main` branch is actively supported for security updates. We recommend always running the latest release of our packages and applications.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | ✅ Yes             |
| < Latest| ❌ No              |

Thank you for helping us keep the Goy Grid secure.
