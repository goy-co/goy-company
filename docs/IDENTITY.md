# Identity Matrix // Hybrid Sovereignty

The Goy Company employs a unique identity model that bridges the gap between traditional Web2 authentication and decentralized Nostr sovereignty.

## The Two Paths

### 1. Portal Access (Traditional)
- **Primary ID**: UUID generated by Better Auth.
- **Verification**: Email and Password.
- **Persistence**: Cloudflare D1 (SQL).
- **Control**: Centralized (The Goy Company manages the vault).
- **Goal**: Accessibility and ease of entry for new entities.

### 2. Sync Access (Sovereign)
- **Primary ID**: 64-character Hex Public Key (NPUB).
- **Verification**: Cryptographic signatures via Browser Extension (NIP-07) or manual NSEC.
- **Persistence**: Decentralized Nostr Relays.
- **Control**: Total Sovereignty (The user owns the private key).
- **Goal**: Absolute independence and censorship resistance.

## The Migration Protocol (The Ritual)
We provide a clear path for entities to transition from Traditional to Sovereign.

1. **Uplink**: User enters via Portal Access.
2. **Initiation**: User triggers "Export to Nostr".
3. **Entropy**: The system generates a new cryptographic keypair.
4. **Broadcast**: Identity metadata is signed and transmitted to the decentralized grid.
5. **Purge**: The central record in the Drizzle/D1 database is wiped.
6. **Rebirth**: The user re-enters the Grid using their new sovereign keys.

## Technical Resolution
The `api-worker` acts as an **Identity Aggregator**. When a request is made for an ID, the worker:
1. Checks if it's a 64-char Hex (Nostr).
2. If not, checks the D1 database for a UUID match.
3. Automatically repairs legacy prefixes (like `GUEST_`) to ensure seamless uplink.
